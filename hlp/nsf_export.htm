NSF export

Important notice: There is a limit on how large NSF's might be, and that is not limited by the tracker itself. 
A track does not necessarily has to fit in a NSF-file, even if it does in the tracker.
To check the avaliable size, create a NSF and check the "Total size"-label in the text window. 
Size may not exceed $8000 (32768) bytes.

To create NSFs, select File/Create NSF to get the NSF export dialog.

The three text-fields are the same as in the tracker main window.

Change Music data address if you want to use music in your NES app. It tells the compiler where the music data is to be stored in the NES's address space. Will not affect NSF's.

Enable bankswitch optimization will remove unused memory by setting the bank bits in the NSF-header. Only usable if DPCM is used. (Could be used to expand the available music and DPCM-space in future versions.)

Force PAL sets the PAL-bit in the NSF-header, should be enabled if you are doing a track for the PAL system. It is automatically checked if the tracker is set to PAL.

Press the Write NSF -button to create a NSF with your track, or Write BIN to create one file with binary music data and one file with DPCM data for use with the music driver when you want music in NES-apps. Finally, there is this option Write PRG that creates a 32 kB file ready to be burned on an EPROM/EEPROM or some other kind of memory for playback on a real NES.
