FamiTracker Help Index

Overview

This is a tracker for producing music for the Famicom/NES PSG chip. It is possible to export to the NSF-format, which allows playback on the real hardware. Currently no extra sound chips are supported.

This software is beta, please report any bugs/comments.

News and bug-fixes in this version

 Added a volume column, this works even if a volume effect setting is used (as opposite to the Exx command)
 Added a play row command (Ctrl+Return)
 Added a keyboard to the instrument settings window (for testing instruments)
 Multiple DPCM samples may be assigned to one instrument
 Track-effect numbers are entered in hexadecimal, this makes it possible to jump more than 99 rows when skipping frame
 A new file format is used for decreased file size (old format is still readable, backups will be created when loaded)
 Fixed a MML parser bug (negative values didn't work)
 Changed the instrument duty cycle / noise mode behavior when no sequence is used (0 is used)
 Sequences failed to loop in some circumstances when compiled into NSF, fixed
 Deleting the instrument number worked in the tracker but failed in compiled NSF, fixed
 Fixed the effect called 'Hi-pitch', it didn't work properly when lowering the pitch in NSF
 NSF hardware sweep is more accurate (should sound as in the tracker)
 and more...
